## 前提与依据

**前提**：所有产品均基于客户**已有云基础设施**（阿里云、腾讯云、华为云、AWS、私有云等）进行部署与集成，不涉足算力、存储等底层资源的售卖。

**文档体系**：01.市场调研 → 02.需求分析 → 03.产品框架；[04.全局优化建议分析](04.全局优化建议分析.md) 为三文档交叉分析产出。

**依据**：本框架综合《01.市场调研》第二章市场规模、第三章通用数据闭环框架、第五章竞品格局、第六章痛点与机会、第九章案例与能力分析，以及《02.需求分析》的 DDD 领域模型（8 个限界上下文、61 条用户故事、功能需求详表、分行业扩展 FD）、形成可执行的产品能力规划。**需求追溯**：能力与模块与 [02.需求分析](02.需求分析.md) 限界上下文（BC）、需求 ID（ING/CAT/ORC/ANN/SIM/BHV/OTA/GOV）一一对应，见 [1.4 能力-模块-BC 与需求追溯](#14-能力模块-bc-与需求追溯)。

**设计原则**：对不同用户匹配不同产品形态（SaaS、私有化、行业包等），但**公共基础能力统一为一套**，产品模块在其上组合配置，实现「一次建设、多端复用」。

## 一、公共基础能力与产品模块识别

### 1.1 公共基础能力清单

以下能力为**所有产品形态共享**，不因行业、部署方式、客户类型而分裂建设。不同产品形态仅是对这些能力的**不同封装与编排**。

| 公共基础能力 | 核心定义 | 能力范围 | 技术栈/典型组件 | 跨行业共性 |
|--------------|----------|----------|-----------------|------------|
| **多源数据接入** | 统一接入多种数据源的标准化能力 | 支持 OSS/S3/COS/Kafka/MQTT/RDS/API 等协议；采集车/量产车/仿真平台/产线 PLC/家居设备等异构源；边缘 Agent 与云端直传两种模式；与客户已有云存储、消息队列的标准对接 | Connector 框架、协议适配器、边缘 SDK、直传/经网关双模式 | 各行业均存在多源、多模态、多系统数据分散；统一接入是打通数据孤岛的前提 |
| **大规模任务调度** | 对数据采集、标注、仿真、训练等任务的全生命周期调度 | 与 K8s、GPU 集群、云函数集成；批处理与流式任务统一编排；优先级、依赖、重试、资源抢占；任务队列与算力池管理 | 任务队列（Redis/Kafka/Celery 等）、K8s Operator、工作流引擎、算力池编排 | 云上效率痛点的核心；智协慧同 Serverless 式跨车云部署 |
| **大规模元数据管理** | 多模态数据的统一目录、血缘与检索 | 支持文本/图像/视频/音频/3D/4D 等 5+ 模态；Omni Catalog 跨源数据目录；与 Flink/Spark/训练框架元数据联动；智能存储与检索加速 | 元数据存储与索引、血缘追踪、与 Hive/Atlas 等兼容或自研 | 通用数据底座；阿里 OpenLake、火山数据湖、百度 DataBuilder 效率+6 倍 |
| **海量数据分析** | 对数据质量、分布、长尾、场景的统计与洞察 | 数据质量评估、标注一致性分析；场景分布统计、长尾样本挖掘；多维度检索与聚合；与标注平台、训练平台的联动分析 | Spark/Flink 等分析引擎、OLAP、统计与聚合服务 | 各行业需场景挖掘、主动采样、长尾检索 |
| **行为与画像分析** | 对「数据主体」行为模式与画像的**抽象建模** | 智驾：驾驶员/路侧/交通参与者行为模式；家居：用户场景意图、设备使用偏好；工业：设备/产线行为、故障模式；具身：动作轨迹、策略泛化模式。偏**实体-行为-画像抽象层**，产出结构化画像与模式 | 画像模型、行为建模框架、实体-行为-画像抽象层 | 主动智能、预测性维护、策略泛化均依赖行为与画像 |
| **数据闭环运营分析** | 对闭环各环节效率、成本、效果的运营级分析 | 采集→标注→训练→回灌各环节的吞吐、周期、成本；模型迭代效果追踪；投入产出比、ROI 分析；与 MLOps 监控、回灌周期对接 | 指标聚合、报表引擎、与 MLOps/监控系统对接 | 智驾需从问题发现到重训部署的周期缩短 |
| **高效数据可视化工具链** | **智能体在环境中的行为分析与定位**——多源传感器、模块输出、决策、感知的**统一可视与时间轴关联** | 视频/IMU/轨迹/模块信号/决策/感知等多维数据时间轴对齐；多路同步播放、轨迹叠加、信号波形、感知框叠加；根因定位、问题样本挖掘；数据质量与运营看板。偏**多维数据呈现与根因定位**，与「行为与画像分析」的抽象建模互补。详见下节 | 多流同步、信号波形、轨迹/点云叠加、React/Vue+ECharts/D3/Three.js；时间轴引擎 | 智驾/具身/工业根因分析、与标注回灌打通 |
| **统一 API 与集成开放** | 对外提供标准接口，支持与训练平台、标注平台、第三方系统对接 | REST/GraphQL 查询与任务下发；Webhook 回调；多语言 SDK；OpenAPI 规范；与训练平台、MES、标注平台的 API 打通 | REST/GraphQL 网关、Webhook 服务、SDK 生成、OpenAPI 文档 | 多源接入偏「拉数进来」，本能力偏「输出与集成」 |
| **数据安全与访问控制** | 统一身份、权限、审计与多租户隔离 | 身份认证（OAuth2/OIDC/SSO）；RBAC 权限；多租户数据隔离；审计日志与访问追踪；敏感数据标记与分级 | 统一认证、RBAC 引擎、审计日志存储、多租户隔离 | 《个保法》《数安法》刚需；合规与治理模块依赖此基础能力 |

**建设原则**：公共基础能力**仅建一套**，共 9 项；通过配置、权限、行业模板等方式适配不同用户，避免按行业或产品线重复建设。

#### 1.1.1 智能体行为分析与可视化（专项）

> **核心场景**：分析与定位**智能体在环境中的行为**。智驾车辆、人形机器人、工业机械臂等智能体在运行中需回溯「为何如此决策」「感知是否准确」「控制是否合理」，问题定位需同时查看各类传感器数据与智能体表现。详见 [01.市场调研](01.市场调研.md) 6.1.7、6.2.9。

| 可视化类别 | 数据/呈现 | 典型场景 |
|------------|-----------|----------|
| **传感器数据** | 视频流、激光雷达、IMU 变化、轮速/转角；多路同步播放 | 环境输入与本体状态 |
| **轨迹与位姿** | 车辆/机器人轨迹、位姿变化、速度曲线；2D/3D 地图叠加 | 实际运动表现 |
| **模块信号输出** | CAN 信号、电机指令、制动/转向；波形图、异常高亮 | 控制层输出 |
| **模块决策** | 规划路径、目标选择、行为意图；与视频/地图叠加 | 决策层输出 |
| **感知输出** | 检测框、分割掩码、BEV 占据、3D 目标；与原始视频/点云叠加 | 感知层输出 |

**能力要求**：多源数据时间轴对齐（毫秒/帧级）；时间轴 scrub 拖拽定位；根因定位与问题样本导出；与标注、仿真、回灌闭环打通。

### 1.2 产品模块识别

产品模块是在公共基础能力之上的**可组合、可配置**的能力单元。不同用户获得不同模块组合，形成差异化产品形态。

| 产品模块 | 依赖的公共能力 | 模块能力 | 适用行业/用户 |
|----------|----------------|----------|---------------|
| **车云编排模块** | 多源接入、任务调度、元数据管理、API 集成、安全与访问控制 | 车端百倍压缩、按规则落盘、分钟级到桌面；Serverless 跨车云；与采集车/量产车深度集成 | 智驾、OEM/车企 |
| **多模态标注模块** | 任务调度、元数据管理、海量数据分析、可视化工具链、API 集成、安全与访问控制 | 2D/3D/4D/BEV 标注；预标注编排云 GPU；质检流水线；标注预览与质检看板；与训练平台 API 对接 | 智驾、具身、工业 |
| **驾驶仿真模块** | 多源接入、任务调度、海量数据分析、可视化、API、安全 | 驾驶仿真、世界模型、极端场景生成；OpenSCENARIO/OpenDRIVE；传感器/动力学/XIL 仿真；仿真回放与结果可视化 | 智驾 |
| **具身/机器人仿真与测试验证模块** | 多源接入、任务调度、海量数据分析、可视化、API、安全 | 物理仿真、Real2Sim 场景重建、动作合成、策略在环验证（XIL）、sim-to-real；成本降 99%+；与标注/训练闭环打通 | 具身智能、人形机器人（P0 核心模块）|
| **跨品牌接入模块** | 多源接入、行为与画像分析、可视化工具链、安全与访问控制 | Matter 协议兼容；跨品牌设备数据汇聚；场景意图解析；用户行为看板 | 智能家居 |
| **预测性维护模块** | 多源接入、行为与画像分析、闭环运营分析、可视化工具链、API 集成、安全与访问控制 | 产线/设备行为分析；故障预测；产线看板与设备监控；与 MES/ERP API 集成 | 工业、物联网 |

**政府/园区能力项细化**（C3）：政务云/信创适配所需公共能力——多源接入（政务数据源）、元数据管理、海量分析、可视化工具链、统一 API、**数据安全与访问控制**（强化）、**合规与治理**（分级分类、审计、数据主权）。不单独建政府模块，通过合规与治理模块 + 部署形态（政务云/信创）适配实现。

**与用户故事对应**：上表客户类型与 [02.需求分析](02.需求分析.md) 第一章用户识别、第二章用户故事大表（61 条）一致；OEM 对应 US-1～24、Tier1 对应 US-25～27a、Robotaxi 对应 US-28～28a、家居对应 US-29～33、具身对应 US-34～40、工业对应 US-41～49、政府对应 US-50～51。

**执行检查点**：每季度末对照本表复盘；Phase 切换需达成该阶段验收标准方可进入下一阶段。

## 七、商业化与风险

### 7.1 定价模式参考

> **E3 竞品定价验证**：下表定价为公开信息与行业估算，**待客户访谈验证**。建议将竞品定价验证纳入 01 调研 7.2/7.3 客户访谈提纲。

| 模式 | 代表 | 适用场景 |
|------|------|----------|
| **按标注量+算力** | 阿里 PAI-iTAG | 智驾标注；包年套餐 |
| **按车/按车型授权** | 智协慧同 | 车云协同；主机厂量产 |
| **按项目/按量** | 星尘、曼孚 | 标注服务；灵活计费 |
| **按存储+计算、API 调用** | 百度 DataBuilder | 大模型/RAG；效率提升变现 |
| **项目制+云端资源** | 腾讯 | 智驾/空间计算 |

**建议**：通用层可采「按量+订阅」；行业包可采「按项目/按车/按工厂」；SaaS 托管采「按存储+API」或「按席位」。

#### 7.1.1 ROI 测算思路

| 测算维度 | 核心公式/思路 | 数据来源 |
|----------|---------------|----------|
| **客户侧 ROI** | 年节省成本 / 年产品投入 ≥ 2x（目标）；智驾可量化传输/存储/标注降本（如智协慧同总成本-85%）| 客户 PoC 前后对比；行业标杆（智协慧同、亮道）|
| **单客户 LTV** | 年合同金额 × 预计续约年数；智驾 OEM 典型 3-5 年、Tier1 1-3 年 | 客户画像 4.1 预算规模；竞品签约周期 |
| **获客成本（CAC）** | 销售+市场投入 / 新签客户数；目标 CAC < LTV/3 | 销售周期、PoC 投入、商务差旅 |
| **盈亏平衡** | 固定成本 /（客单价 - 单客户变动成本）；通用层边际成本低，重点控研发与获客 | 团队规模、云资源、交付人天 |

**阶段目标**（供 Phase 2-4 参考）：Phase 2 实现单客户为正毛利；Phase 3 覆盖获客成本；Phase 4 追求规模经济。

#### 7.1.2 获客与留存考量

| 考量项 | 建议 | 依据 |
|--------|------|------|
| **获客渠道** | 技术选型期切入（PoC 前）；年度预算季（Q4 立项）；新车型/新项目规划期 | 4.2 采购决策链与关键触点 |
| **留存抓手** | 私有化深度集成、数据迁移成本、与客户云/训练平台 API 绑定 | 切换成本高则续约率提升 |
| **扩张路径** | 单主机厂→多车型；单工厂→集团推广；标注→车云编排→全链路 | 模块组合可渐进扩展 |

### 7.2 风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| **L3 法规延期** | 智驾商业化节奏放缓 | 跟踪政策节奏；**季度检查点**：若延期超 6 个月触发工业/具身优先级上调；L2+ 市场已足够大（82.6% 新能源渗透）；合规工具先行；拓展工业/具身对冲 |
| **数据出境限制收紧** | 跨境客户、出海企业受限 | 本地化部署能力；脱敏与备案支持；工业数据空间适配；明确不出境承诺 |
| **云厂商向下游延伸** | 挤压工具链空间、价格竞争 | 聚焦垂直深度（车云编排、4D 标注）；与云厂商合作而非对抗；工业/家居等云厂商覆盖弱的领域 |
| **竞品定价战** | 毛利承压、获客成本上升 | 差异化能力（降本效果量化）；客户锁定（私有化、深度集成）；多行业分散风险 |
| **技术/算法依赖** | 预标注、仿真等依赖大模型/物理引擎，供应或迭代不及预期 | 多引擎兼容（兼容开源+商业）；预留替换接口；核心能力自研与外包平衡 |
| **关键人才流失** | 产品、算法、交付核心人员流动 | 文档化、知识沉淀；核心模块多人备份；激励与股权绑定 |
| **客户账期与回款** | OEM/工业项目账期长，现金流压力 | 分阶段付款（签约/验收/上线）；优先拓展 Tier1/中小客户改善周转 |
| **行业周期与预算收缩** | 智驾融资降温、工业数字化预算削减 | 多行业分散；轻量级/按需订阅产品线；与平台商合作分担获客成本 |
| **首款产品选型偏差** | 切入行业或模块与市场不匹配 | Phase 1 充分调研；Q1 行业选定需多方案对比；保留 1 个备选方向 |
