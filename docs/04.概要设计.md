## 一、架构总览

### 1.1 系统分层架构

基于 [02.需求分析](02.需求分析.md) 的限界上下文划分与 [03.产品框架](03.产品框架.md) 的公共能力设计，技术架构采用**分层 + 限界上下文**模式：

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                              表现层 / 开放层                                           │
│  Web 控制台 │ REST/GraphQL API │ Webhook │ 多语言 SDK │ OpenAPI 规范                   │
└─────────────────────────────────────────────────────────────────────────────────────┘
                                        │
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                              应用层 / 领域服务层                                        │
│  BC-ING │ BC-CAT │ BC-ORC │ BC-ANN │ BC-SIM │ BC-BHV │ BC-OTA │ BC-GOV               │
│  （各 BC 应用服务，领域逻辑封装）                                                        │
└─────────────────────────────────────────────────────────────────────────────────────┘
                                        │
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                              领域事件 / 消息层                                          │
│  事件总线（Kafka / 云消息）│ 领域事件发布订阅 │ 跨 BC 解耦                               │
└─────────────────────────────────────────────────────────────────────────────────────┘
                                        │
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                              基础设施层                                                 │
│  客户 OSS/S3/COS │ Kafka │ K8s │ 对象存储 │ 关系/文档DB │ Redis │ GPU 集群             │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

**设计原则**：
- **BC 独立部署**：各限界上下文可独立编译、部署、扩缩容
- **事件驱动协作**：跨 BC 协作以领域事件为主，避免强耦合
- **统一开放**：对外统一 API 网关，内部按 BC 拆分微服务

### 1.2 限界上下文与技术模块映射

| 限界上下文 | 技术模块名称 | 技术职责 | 建议技术形态 |
|------------|---------------|----------|--------------|
| **BC-ING** | ingestion-service | Connector 框架、协议适配、边缘 Agent | 微服务 + 边端 SDK |
| **BC-CAT** | catalog-service | 元数据存储、检索、血缘、场景挖掘 | 微服务 + 检索引擎 |
| **BC-ORC** | orchestration-service | 任务调度、车云编排、成本看板 | 微服务 + K8s Operator |
| **BC-ANN** | annotation-service | 标注工作台、预标注、质检流水线 | 微服务 + Web 前端 |
| **BC-SIM** | simulation-service | 驾驶/具身仿真、仿真任务调度 | 微服务 + 仿真引擎 |
| **BC-BHV** | behavior-service | 行为回放、时间轴对齐、根因定位 | 微服务 + 可视化前端 |
| **BC-OTA** | ota-service | OTA 发布、灰度、回滚、设备对接 | 微服务 |
| **BC-GOV** | governance-service | 脱敏、审计、多租户、分级分类 | 微服务 + 横切中间件 |

### 1.3 核心数据流与集成拓扑

```
[ 外部数据源 ]                     [ 闭环内部 ]
  OSS/Kafka/PLC/Matter     ──►  BC-ING (接入)
 采集车/产线/家居设备                   │
                                       ▼
  K8s/GPU 集群              ◄──  BC-ORC (编排)  ◄──  BC-CAT (元数据)
                                       │
                    ┌─────────────────┼─────────────────┐
                    ▼                 ▼                 ▼
              BC-ANN (标注)    BC-SIM (仿真)      BC-BHV (行为)
                    │                 │                 │
                    └────────────────┼─────────────────┘
                                      │
                              [ 训练平台 ]（外部）
                                      │
                                      ▼
                              BC-OTA (部署) ──►  车端/机器人/产线设备
                                      │
                                      ▼
                              BC-GOV (横切：审计、脱敏、多租户)
```

**集成顺序**（与 [03.产品框架](03.产品框架.md) 1.6 一致）：
- Layer 0：BC-ING、BC-GOV
- Layer 1：BC-CAT、BC-ORC
- Layer 2：BC-ANN、BC-SIM
- Layer 3：BC-BHV、BC-OTA

## 二、技术选型

> **详细选型分析**：语言、数据库、数据湖、OLAP、可视化等完整选型论证见 [06.技术选型](06.技术选型.md)。

### 2.1 技术选型总览

| 分层/能力 | 建议技术 | 备选 | 说明 |
|-----------|----------|------|------|
| **运行时** | Kubernetes | 云厂商托管 K8s | 与客户已有云对接；支持阿里 ACK、腾讯 TKE、华为 CCE |
| **服务框架** | Go / Java (Spring Boot) | — | Go 适合高并发/边端；Java 适合企业级集成 |
| **API 网关** | Kong / APISIX / 云厂商 API 网关 | — | 统一入口、鉴权、限流 |
| **消息/事件** | Kafka | 云 Kafka / RocketMQ | 领域事件、车云数据流；支持客户已有 Kafka |
| **元数据存储** | PostgreSQL + Elasticsearch | MySQL + 自研索引 | 元数据关系存储 + 检索加速 |
| **对象存储** | 客户 OSS/S3/COS | — | 不自建，对接客户已有存储 |
| **任务调度** | K8s Job/CronJob + 工作流引擎 | Argo Workflows / Airflow | DAG 编排、批处理 |
| **预标注/仿真** | GPU 集群 + 容器化 | 云 GPU | 调用客户 K8s GPU 节点 |
| **前端** | React / Vue 3 + TypeScript | — | 标注工作台、行为回放、管理控制台 |
| **边缘 Agent** | Rust / Go 轻量运行时 | — | 车端/网关部署；低资源占用 |

### 2.2 选型依据与约束

| 约束 | 说明 | 对选型的影响 |
|------|------|--------------|
| **基于客户云** | 不涉足算力、存储售卖 | 对接 OSS/Kafka/K8s 等标准接口；支持多云 |
| **私有化部署** | Tier1、政府等数据不出域 | 支持容器化部署、无强云依赖 |
| **多行业** | 智驾、工业、家居、具身 | 协议可扩展；行业包可插拔 |
| **合规** | 《个保法》《数安法》 | 审计、脱敏、多租户必选 |

## 三、各 BC 技术方案

### 3.1 BC-ING 数据接入

| 维度 | 技术方案 |
|------|----------|
| **Connector 框架** | 插件化 Connector；每种协议独立模块；配置驱动 |
| **支持协议** | OSS/S3/COS（对象存储）、Kafka/MQTT（消息）、RDS/API（增量拉取）、PLC/MES（工业）、Matter（家居）|
| **边缘 Agent** | 轻量二进制（Rust/Go）；本地压缩（目标 ≥50x）；可配置过滤规则；支持直传 OSS 与经网关汇聚 |
| **数据流出** | 接入完成发布 `DataIngestionCompleted` 事件；触发 BC-CAT 注册、BC-ORC 编排 |
| **扩展性** | 新增协议通过实现 Connector 接口 + 配置注册 |

**核心组件**：
- Connector 注册中心
- 任务调度器（拉取/订阅）
- 边缘 Agent SDK（车端/工业网关）

### 3.2 BC-CAT 数据目录与元数据

| 维度 | 技术方案 |
|------|----------|
| **元数据存储** | PostgreSQL 存储关系数据；Elasticsearch 做检索索引 |
| **多模态模型** | 支持文本/图像/视频/音频/3D/4D；Schema 可扩展 |
| **检索性能** | 百万级 P95 < 500ms；条件检索、聚合、分页 |
| **血缘** | 有向图存储；支持采集→标注→训练 lineage |
| **场景挖掘** | 调用分析服务（Spark/Flink 或轻量聚合）；产出场景类型、长尾标签 |
| **对外接口** | REST API 查询；支持与 Flink/Spark/训练框架元数据联动 |

**核心组件**：
- 元数据服务（CRUD、检索）
- 血缘服务
- 场景挖掘服务（可选独立部署）

### 3.3 BC-ORC 任务编排

| 维度 | 技术方案 |
|------|----------|
| **调度引擎** | K8s 原生 Job/CronJob + 自研 DAG 编排层；或 Argo Workflows |
| **车云编排** | 规则引擎配置（触发条件、压缩策略、落盘路径）；车端 Agent 与云编排协同 |
| **批处理与流式** | 批处理：定时/触发；流式：Kafka 消费后触发 |
| **成本运营看板** | 调用成本核算领域服务；指标聚合；前端展示 |
| **依赖与重试** | DAG 依赖；失败重试与退避策略 |

**核心组件**：
- 编排引擎（DAG 解析、任务下发）
- K8s Operator（可选，深度集成）
- 车云规则配置服务
- 成本核算服务

### 3.4 BC-ANN 标注与质检

| 维度 | 技术方案 |
|------|----------|
| **标注工作台** | Web 前端（React/Vue）；多模态视图（2D/3D/BEV/4D）；时间轴联动 |
| **预标注** | 调用 K8s GPU Job；支持多模型串行/并行；模型仓库管理 |
| **质检流水线** | 多级队列（初检/复检/抽检）；可配置通过阈值；状态机驱动 |
| **导出** | REST API 导出 COCO/KITTI/自定义格式；Webhook 通知训练平台 |
| **与训练平台对接** | 标准格式 + 可选 MLOps 对接（MLflow 等）|

**核心组件**：
- 标注任务服务
- 预标注调度服务
- 质检流水线服务
- 标注前端应用

### 3.5 BC-SIM 仿真与合成

| 维度 | 技术方案 |
|------|----------|
| **驾驶仿真** | OpenSCENARIO/OpenDRIVE 兼容；世界模型集成；极端场景生成 |
| **具身仿真** | MuJoCo/Isaac 或兼容物理引擎；Real2Sim 场景重建；XIL 验证 |
| **任务调度** | K8s GPU 调度；批量仿真；日产能目标 ≥ 5,000 条/集群 |
| **产出** | 仿真数据注册至 BC-CAT；可进入 BC-ANN 标注流水线 |
| **成本控制** | 单条 ≤ 0.05 元；资源复用与调度优化 |

**核心组件**：
- 仿真引擎（按行业选型：驾驶/具身）
- 仿真任务调度服务
- 场景配置与参数服务

### 3.6 BC-BHV 行为分析与回灌

| 维度 | 技术方案 |
|------|----------|
| **多维时间轴** | 视频/IMU/轨迹/CAN/PLC/传感器/感知/决策统一时间轴；毫秒/帧级对齐 |
| **前端技术** | React/Vue + ECharts/D3/Three.js；多流同步、波形、叠加 |
| **数据源适配** | 车端（视频+IMU+轨迹+模块信号）、产线（PLC+传感器+视频）、机器人（遥操作+仿真）|
| **问题样本** | 标记、筛选、导出至 BC-ANN 或触发 BC-ORC 回灌 |
| **根因定位** | 感知/决策/控制/设备分层展示 |

**核心组件**：
- 行为会话服务（数据加载、对齐）
- 回放前端应用
- 问题样本导出服务

### 3.7 BC-OTA OTA 与模型部署

| 维度 | 技术方案 |
|------|----------|
| **发布类型** | SOTA/FOTA/DOTA；支持模型/固件差分更新 |
| **与训练平台对接** | 接收「模型就绪」Webhook；版本管理；灰度策略输入 |
| **灰度与回滚** | 分批推送；可回滚至上一版本；状态持久化 |
| **设备协议** | 标准 HTTP/OTA 协议；与车端/机器人/产线设备对接；安全校验 |
| **契约** | 模型就绪事件格式：versionId、artifactUri、grayPolicy |

**核心组件**：
- OTA 发布服务
- 训练平台对接适配器
- 设备通信模块（协议可插拔）

### 3.8 BC-GOV 合规与治理

| 维度 | 技术方案 |
|------|----------|
| **脱敏** | 脱敏引擎（泛化、加密、差分隐私等）；可配置规则；可测场景覆盖 |
| **审计** | 关键操作全量记录；不可篡改存储；可检索、可导出 |
| **多租户** | 租户 ID 贯穿全链路；数据隔离；RBAC 按租户 |
| **部署拓扑** | SaaS/私有化/边缘；拓扑配置驱动多租户与合规策略 |
| **政务云/信创** | 兼容信创组件（数据库、中间件）；部署形态可选 |

**核心组件**：
- 审计日志服务（横切）
- 脱敏引擎
- 多租户上下文中间件

## 四、数据架构

### 4.1 元数据模型

**核心实体**（与 [02.需求分析](02.需求分析.md) 6.2 对齐）：

| 实体 | 主要属性 | 存储 |
|------|----------|------|
| DataAsset | assetId, storagePath, modality, sourceId, tenantId | PostgreSQL |
| MetadataEntry | schema, lineageRefs | PostgreSQL + ES 索引 |
| AnnotationResult | taskId, assetId, bbox/mask/label | PostgreSQL |
| BehaviorSession | sessionId, timeRange, dataSourceRefs | PostgreSQL |
| ReleaseTask | taskId, targetDevices, grayPolicy | PostgreSQL |

**多模态 Schema**：JSON Schema 可扩展；支持文本/图像/视频/音频/3D/4D 属性定义。

### 4.2 领域事件存储与消息

| 用途 | 技术方案 |
|------|----------|
| **事件传输** | Kafka Topic 按事件类型或 BC 分区 |
| **事件格式** | JSON Schema；含 eventId、timestamp、payload、tenantId |
| **顺序保证** | 同一聚合根的事件按 partition key 保序 |
| **回溯** | 可选 Event Sourcing 存储；默认事件日志 + 状态持久化 |

**核心领域事件**（见 [02.需求分析](02.需求分析.md) 附录 A）：
- DataIngestionCompleted、MetadataRegistered、AnnotationApproved
- ProblemSampleIdentified、FeedbackDecisionMade、SimulationDataGenerated
- ReleaseTaskCreated、DeploymentBatchCompleted、AuditRecordCreated

## 五、集成与接口设计

### 5.1 跨 BC 集成方式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **领域事件** | 异步；发布-订阅 | 接入完成→注册元数据；标注通过→更新血缘；问题样本→触发回灌 |
| **REST 调用** | 同步；上游调下游 | 查询元数据；创建标注任务 |
| **OHS（开放主机服务）** | 对外标准 API | 各 BC 对外暴露 OpenAPI |

**契约管理**：领域事件 JSON Schema 版本化；API 版本路由（/v1/、/v2/）。

### 5.2 外部系统接口契约

| 外部系统 | 接口类型 | 契约要点 |
|----------|----------|----------|
| **客户 OSS/Kafka** | Connector 拉取 | OSS/S3/COS 列出、读取、元数据；Kafka 订阅、offset 提交 |
| **训练平台（MLOps）** | 入：标注导出 API；出：模型就绪 Webhook | 标注格式 COCO/KITTI/自定义；模型就绪含 versionId、artifactUri、grayPolicy |
| **车端/机器人/产线** | OTA 下发 | HTTP/OTA 协议；差分包格式；安全校验 |
| **K8s** | 任务调度 | Job/CronJob CRD；资源请求 CPU/内存/GPU |

## 六、部署架构

### 6.1 部署形态

| 形态 | 说明 | 适用客户 |
|------|------|----------|
| **SaaS** | 托管于客户云账户；多租户 | 中小客户、快速试用 |
| **私有化** | 部署至客户 VPC/机房；单租户或有限多租户 | OEM、Tier1、政府 |
| **边缘 Agent** | 轻量二进制部署车端/工业网关/家庭网关 | 智驾、工业、家居 |

**容器化**：各 BC 微服务容器化；Helm Chart 或 Kustomize 管理；支持客户 K8s 集群。

### 6.2 多租户与隔离

| 层级 | 隔离方式 |
|------|----------|
| **数据** | tenantId 贯穿表设计；查询强制过滤 |
| **计算** | 可选 Namespace 隔离；资源配额 |
| **网络** | 可选 VPC 隔离（私有化）|
| **认证** | OAuth2/OIDC；SSO；租户上下文注入 |

## 七、非功能设计

### 7.1 性能设计

| 指标 | 目标 | 设计措施 |
|------|------|----------|
| **元数据检索** | 百万级 P95 < 500ms | ES 索引；条件优化；分页 |
| **任务调度** | ≥ 10,000 任务/天 | 批处理优化；队列削峰 |
| **车端压缩** | ≥ 50x | 压缩算法；按规则过滤 |
| **端到端到桌** | 车云 < 10 分钟 | 并行传输；规则落盘优化 |
| **预标注** | 单帧较人工降 50%+ | GPU 批处理；模型优化 |
| **具身仿真** | 单条 ≤ 0.05 元；≥ 5,000 条/天 | 资源复用；批量调度 |

### 7.2 安全设计

| 能力 | 设计措施 |
|------|----------|
| **认证** | OAuth2/OIDC/SSO；JWT 或 Session |
| **授权** | RBAC；租户隔离 |
| **审计** | 100% 覆盖；不可篡改 |
| **脱敏** | 可配置规则；多级脱敏 |
| **传输** | TLS；边云通信加密 |
| **数据出境** | 本地化优先；备案支持 |

### 7.3 可观测性设计

| 维度 | 设计措施 |
|------|----------|
| **日志** | 结构化日志；集中采集（ELK/Loki）；按 BC 标签 |
| **指标** | Prometheus + Grafana；各 BC 暴露 metrics |
| **链路** | OpenTelemetry；跨 BC  Trace |
| **告警** | 关键指标阈值；与运维平台对接 |
| **闭环效率** | 吞吐、延迟、回灌周期；成本核算看板 |

**回灌周期**（与 [99.全局优化建议分析](99.全局优化建议分析.md) G1 对齐）：定义「问题样本识别→标注→训练→OTA」端到端天数为闭环效率核心 KPI；在 ORC/成本核算中可观测。

## 八、技术风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| **训练平台多样性** | 对接成本高；格式不兼容 | 优先支持 2-3 种主流格式；与 1-2 家云厂商 MLOps 认证对接 |
| **OTA 与训练平台契约未定** | 闭环断裂 | 明确模型就绪事件格式；纳入 OTA 模块设计（与 G2 对齐）|
| **编排中枢单点** | ORC 故障全链阻塞 | 编排规则可观测、可回滚；高可用部署 |
| **预标注/仿真依赖大模型/物理引擎** | 供应或迭代不及预期 | 多引擎兼容；预留替换接口 |
| **边缘 Agent 资源受限** | 车端/网关性能不足 | 轻量化设计；Rust/Go；可配置功能裁剪 |
| **多云对接复杂度** | 阿里/腾讯/华为等差异 | 抽象适配层；Connector 按云厂商封装 |
